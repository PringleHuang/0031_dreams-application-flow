階段一 - 問卷續約/新約分流

客戶收到問卷通知後, 問卷第一步詢問是否為續約案件。續約僅填電號走簡化流程, 新約需完整資料。
分類邏輯：問卷 Webhook 來自 work-survey/7, payload 不含案件狀態。下游 Lambda 從 DREAMS_APPLY_ID（格式: {出貨單號}-{ragicId}）split 取最後一段作為案件 ragicId, 再查詢 business-process2/2/{ragicId} 取得案件類型。

新約流程 - 資訊收集階段

新約案件 - 從 DREAMS_APPLY_ID 解析目標記錄 ID, AI 逐份比對佐證文件, 一次性將所有結果寫入 RAGIC, 公司聯絡人在 RAGIC 上確認並手動改狀態。
補件問卷分類：補件問卷來自 work-survey/9, 下游 Lambda 從 DREAMS_APPLY_ID 解析 ragicId → 查詢案件管理表 1015456 欄位 → 區分「資訊補件」或「台電補件」。

新約流程 - 台電審核階段

人工確認OK後, 呼叫 DREAMS 填單 API 取得案號與 PDF, 連同佐證文件發送給台電。台電回覆後 AI 語意分析寫入結果, 進入「發送前人工確認」由人工決定下一步。

新約流程 - 安裝和結案

人工確認核准後通知客戶安裝資料收集器, 客服協助 DREAMS 自主檢查與上線, 最後同步至 SunVeillance 並結案。

續約簡化流程

續約案件不需 AI 判定和台電審核。客戶填電號後登入 SunVeillance 選擇續約案場, 完成後回寫雲端 RAGIC 即結案。
flowchart TD A["案件狀態 - 續約處理"] --> B["提供 SunVeillance 登入資訊給客戶"] B --> C["客戶登入 SunVeillance"] C --> D["客戶選擇續約案場"] D --> E{"選擇完成"} E -->|否| D E -->|是| F["回寫雲端 RAGIC 案件管理表單"] F --> G["案件狀態 → 已結案"]

案件狀態轉換總覽

案件共 11 個狀態, 2 種類型。續約走簡化路徑, 新約走完整路徑。台電回覆後需經「發送前人工確認」再決定下一步。

狀態說明

新開案件 - 出貨掃描服務建立, 等待發送問卷
待填問卷 - 問卷通知已發送, 等待客戶填寫
續約處理 - 等待客戶選擇續約案場
待人工確認 - AI判定完成, 等待人工在RAGIC改狀態
資訊補件 - 等待客戶補件
台電審核 - 已送出審核申請, 等待台電回覆
發送前人工確認 - 台電回覆已分析, 等待人工確認
台電補件 - 台電駁回, 等待客戶補件
安裝階段 - 等待客戶安裝設備
完成上線 - DREAMS已上線, 同步中
已結案 - 案件關閉
